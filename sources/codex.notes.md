# Codex Source Notes

Watch Codex as provider-native frontier capability. The key question is not
"did OpenAI ship a feature?" The key question is whether Bitter should route
through, constrain, receipt, test, or stop mirroring that capability.

High-signal areas:

- goals, long-horizon work, workflows, and automations
- subagents, skills, plugins, hooks, and MCP surfaces
- sandboxing, approvals, permissions, and enterprise governance
- `AGENTS.md`, rules, and local context behavior
- browser, computer-use, local-environment, and remote-environment changes
- CLI non-interactive mode, SDKs, GitHub integrations, and app server surfaces
- usage, pricing, model, and capability changes that affect run contracts

Do not over-weight:

- generic model announcements without Codex operator impact
- third-party commentary without a primary Codex source
- benchmark claims unless they include method and reproducible context
- provider marketing copy that does not change Bitter action
